Sentinel — Oversight

Policy enforcement, Titan routing, MOD-facing oversight. All responses route through Sentinel.

Control flow

User → Core → Sentinel → (optional Titan) → Sentinel → Core → User.

Sentinel cannot be bypassed. Policy rules and Titan invocation are applied here.

  • SentinelOverlay.process — Classifies request (high-risk / operational / informational), runs SentinelRules, sets riskFlags, uncertaintyLabel, recommendedNextSteps.
  • Titan — Invoked only when sentinel sets invokeTitan (e.g. high-risk or uncertainty requested). Advisory only.
  • Audit — Every request logs: request_received, sentinel_processed, titan_invoked (when applicable), response_sent.

Policy (SentinelRules)

Outputs are advisory, not commanding. Uncertainty explicitly labeled.

RuleBehaviour
Illegal / harmfulBlocked; warnings logged.
Autonomy / commandsrequiresHuman; responseFrame = advisory_only; no autonomous execution.
Certainty claimsConfidence labeled; no guarantees without evidence.
Response frameAdvisory only; outcomes require human approval.
UncertaintyriskFlags, uncertaintyLabel (high/medium/low), recommendedNextSteps.
